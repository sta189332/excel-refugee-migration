The Master Excel-to-R Cheat Sheet

You do not need to memorize everything in R.

You need a reliable bridge.

That is what this appendix is for. Keep it open during reporting work. Print it. Mark it up. Return to it when Excel habits are faster than confidence and the deadline is loud enough to make old workflows look tempting again.

The point here is not conceptual elegance.

The point is speed.

Use this appendix when the file is messy, the request is urgent, and your brain needs a clean translation path before 9:00 AM.

TipCheat Sheet Box

The fastest way to use this appendix is simple.

  1. identify the Excel task you are about to do
  2. find the matching row in the translation map
  3. copy the R pattern, not the fear
  4. save the script, even if the output still goes back to Excel

A.1 Core Translation Map

The table below is the working dictionary.

Do not treat it like theory. Treat it like operational translation. The question is not “What is the R equivalent in an abstract sense?” The question is “What do I do instead of the thing I keep doing manually in Excel?”

Excel Task Common Excel Method R Equivalent Why the R Version Wins Example Workflow
Add a total SUM(range) summarise(total = sum(value, na.rm = TRUE)) The logic is visible, reusable, and easy to rerun on new data without touching cells again. Total monthly sales across all regions after importing sales_actuals.csv.
Average a field AVERAGE(range) summarise(avg_value = mean(value, na.rm = TRUE)) Handles missing values explicitly and keeps the calculation tied to the dataset instead of a fragile range. Average payroll by department after cleaning blanks.
Count records meeting a condition COUNTIF(range, criteria) filter(condition) |> summarise(n = n()) The condition is readable and easy to adjust when the rule changes. Count employees marked "Active" in an HR extract.
Sum by one condition SUMIF(range, criteria, sum_range) filter(condition) |> summarise(total = sum(value, na.rm = TRUE)) Separates filtering from calculation, which makes the business rule much easier to inspect. Sum sales for the West region only.
Sum by multiple conditions SUMIFS() filter(cond1, cond2, cond3) |> summarise(total = sum(value, na.rm = TRUE)) Multiple rules stay readable instead of getting buried inside formula punctuation. Sum March revenue for Region == "West" and Product_Line == "Core".
One-condition logic IF(test, yes, no) mutate(flag = if_else(test, yes, no)) Applies the rule across a whole column at once and makes it easier to audit later. Flag overdue invoices as "Overdue" or "Current".
Multi-condition logic nested IF() mutate(group = case_when(...)) Complex classification stops looking like a formula accident and starts looking like declared business logic. Classify employees as Core Staff, Flexible Staff, or Contractors.
Hide errors IFERROR(formula, value) coalesce(), replace_na(), or guarded logic with if_else() Makes the missing or failing case explicit instead of hiding it behind a silent rescue formula. Replace missing target values with zero before variance calculations.
Trim spaces TRIM(cell) mutate(field = str_trim(field)) Cleans the whole column at once, including fields imported from ugly exports. Remove trailing spaces from region, department, or employee_id.
Join text together CONCATENATE() or & mutate(full_name = str_c(first_name, last_name, sep = " ")) Cleaner syntax, easier to reuse, and better for large tables. Build a full employee name for reporting output.
Pull left, right, or middle text LEFT(), RIGHT(), MID() str_sub() String extraction becomes readable and consistent across long columns. Pull a cost-center prefix from an account code.
Find text inside text FIND() or SEARCH() str_detect(), str_locate() Easier for pattern checks and more scalable when the logic must be reused later. Detect whether a product description contains "Premium".
Format text or dates TEXT(value, format) format() or str_c() with cleaned fields Keeps formatting separate from the raw value so the underlying data stays usable. Format a report month as "Jan 2026" for a slide label.
Remove duplicates Data tab → Remove Duplicates distinct(key, .keep_all = TRUE) You define what counts as duplication and keep the rule in the workflow. Keep one row per employee_id and pay_period.
Sort data Sort buttons arrange(field) or arrange(desc(field)) Sorting becomes part of the logic, not an interface state that gets lost next week. Rank regions from highest to lowest revenue.
Filter rows Drop-down filter filter(condition) The filter rule becomes readable and repeatable instead of hidden in the workbook view. Keep only "Active" employees or only "West" sales.
Conditional classification helper columns plus IF chains mutate() plus case_when() Lets you create professional classification logic without dragging formulas down forever. Group cost centers into strategic clusters.
Lookup one table into another VLOOKUP() left_join() Joins are easier to audit, more stable across changing column order, and far less brittle than lookups. Bring payroll values into a headcount file by employee_id.
Lookup with flexible column position XLOOKUP() left_join() plus select() The relationship is based on keys, not where the source column happens to sit. Match target values to actuals by region and month.
Flexible two-way match INDEX() plus MATCH() left_join(), filter(), or grouped summaries depending on the real task Usually reveals that the underlying problem is relational, not cell-based. Pull product-line names into sales actuals using product_code.
Summarize large tables Pivot Table group_by() plus summarise() Summary logic becomes visible, rerunnable, and much easier to verify. Headcount by department or revenue by region and month.
Split text into separate fields Text to Columns separate() The split rule becomes reusable instead of being performed once and forgotten. Split "Region - Department" into two columns.
Pattern-based fill Flash Fill mutate() with str_*() functions You stop relying on Excel’s guess and start declaring the logic directly. Extract initials, clean codes, or standardize labels.
Fix ugly dates manual formatting, helper columns, trial and error mdy(), dmy(), ymd(), floor_date() Date parsing becomes explicit, which reduces false trends and silent errors. Convert 04/01/2026 or 1-Apr-26 into real dates.
Handle missing values blank cells, manual fill, ad hoc replacements replace_na() or coalesce() Makes the treatment of blanks visible and consistent. Replace missing bonus values with zero and missing managers with "Unassigned".
Detect duplicates manual scan, conditional formatting count(key) |> filter(n > 1) Finds repeated records quickly and helps you decide whether they are errors or legitimate repeats. Check whether employee_id appears more than once in a supposedly unique file.
Export cleaned output Save As, copy into new workbook write_csv() or write.xlsx() The cleaned result becomes reproducible and easier to hand over without workbook archaeology. Save a cleaned payroll file for finance or export a board-ready summary workbook.

Quick Syntax Shelf

Use these as first-response patterns when the task is obvious and the deadline is not interested in your emotions.

# Import a CSV
sales <- read_csv("inputs/sales_actuals.csv")

# Import an Excel sheet
targets <- read_excel("inputs/sales_targets.xlsx")

# Clean names
sales <- sales |> clean_names()

# Trim ugly text
sales <- sales |> mutate(region = str_trim(region))

# Replace missing values
sales <- sales |> mutate(sales_value = replace_na(sales_value, 0))

# Join reference data
sales_enriched <- sales |> left_join(targets, by = c("region", "month"))

# Summarise
summary_tbl <- sales_enriched |>
  group_by(region) |>
  summarise(total_sales = sum(sales_value, na.rm = TRUE), .groups = "drop")

# Remove duplicates
unique_tbl <- sales_enriched |> distinct(employee_id, pay_period, .keep_all = TRUE)

# Export CSV
write_csv(summary_tbl, "outputs/summary_tbl.csv")

# Export Excel workbook
write.xlsx(summary_tbl, "outputs/summary_tbl.xlsx")
WarningQuick Warning

The old trap is not that Excel tasks are impossible.

The old trap is that the same task gets solved from scratch every week with no memory except your own.

If the rule can be written once, it should not be rebuilt by hand forever.

A.2 Common Patterns

These are not polished case studies. They are repeatable field recipes. Use them when you already know the shape of the request and need a clean route from messy file to stable output.

Monthly Sales Reporting Pattern

Situation
You receive actual sales as a CSV, targets in Excel, and a product or region lookup table from another team. Leadership wants a monthly view by region with one clear summary and one chart.

What Usually Happens In Excel
You copy each file into one workbook, repair labels, build lookups, refresh a pivot, fix the sort order, and wonder whether the totals still match the original export.

Cleaner R Workflow
Import each file. Standardize names. Clean region labels. Parse the month field. Join target and lookup data. Summarize by region and month. Export the summary and the chart.

Recommended Core Functions
read_csv(), read_excel(), clean_names(), str_trim(), left_join(), group_by(), summarise(), write_csv()

Use this when the question is simple but the file chain is not. The real value here is not the formula replacement. It is that actuals, targets, and lookup logic all get pulled into one readable sequence.

library(readr)
library(readxl)
library(dplyr)
library(stringr)
library(lubridate)
library(janitor)

sales_actuals <- read_csv("inputs/sales_actuals.csv") |>
  clean_names() |>
  mutate(
    region = str_trim(region),
    region = str_to_title(region),
    sale_month = floor_date(mdy(sale_date), unit = "month"),
    sales_value = replace_na(sales_value, 0)
  )

sales_targets <- read_excel("inputs/sales_targets.xlsx", sheet = "Targets") |>
  clean_names() |>
  mutate(
    region = str_trim(region),
    region = str_to_title(region),
    sale_month = floor_date(mdy(month), unit = "month"),
    target_value = replace_na(target_value, 0)
  )

product_lookup <- read_csv("inputs/product_lookup.csv") |>
  clean_names() |>
  mutate(product_code = str_trim(product_code)) |>
  distinct(product_code, .keep_all = TRUE)

monthly_sales_summary <- sales_actuals |>
  left_join(product_lookup, by = "product_code") |>
  group_by(sale_month, region) |>
  summarise(
    actual_sales = sum(sales_value, na.rm = TRUE),
    .groups = "drop"
  ) |>
  left_join(
    sales_targets |>
      group_by(sale_month, region) |>
      summarise(target_sales = sum(target_value, na.rm = TRUE), .groups = "drop"),
    by = c("sale_month", "region")
  ) |>
  mutate(
    target_sales = replace_na(target_sales, 0),
    variance = actual_sales - target_sales
  ) |>
  arrange(sale_month, region)

write_csv(monthly_sales_summary, "outputs/monthly_sales_summary.csv")

HR Headcount Reconciliation Pattern

Situation
You receive one headcount extract from HR and another staffing file from payroll or workforce planning. The task is to reconcile counts by department and identify mismatches or missing categories.

What Usually Happens In Excel
You sort by employee ID, highlight duplicates, repair names manually, use a lookup to pull fields across, then spend too long checking whether the mismatches are real or workbook-induced.

Cleaner R Workflow
Import both tables. Clean names. Trim IDs and department labels. Remove duplicate rows by a defined key. Join staffing attributes into the headcount table. Group by department. Compare counts.

Recommended Core Functions
read_csv(), clean_names(), str_trim(), distinct(), left_join(), group_by(), summarise()

This workflow is less about clever code than about calming suspicion. You need one table that tells you what HR says, what payroll says, and where the counts drift apart.

library(readr)
library(dplyr)
library(stringr)
library(janitor)

headcount_hr <- read_csv("inputs/hr_headcount.csv") |>
  clean_names() |>
  mutate(
    employee_id = str_trim(employee_id),
    department = str_to_title(str_trim(department))
  ) |>
  distinct(employee_id, .keep_all = TRUE)

staffing_reference <- read_csv("inputs/payroll_staffing_reference.csv") |>
  clean_names() |>
  mutate(
    employee_id = str_trim(employee_id),
    payroll_department = str_to_title(str_trim(payroll_department))
  ) |>
  distinct(employee_id, .keep_all = TRUE)

headcount_reconciliation <- headcount_hr |>
  left_join(staffing_reference, by = "employee_id") |>
  mutate(
    department_match = if_else(department == payroll_department, "Match", "Mismatch")
  )

department_reconciliation_summary <- headcount_reconciliation |>
  group_by(department) |>
  summarise(
    hr_count = n(),
    payroll_count = sum(!is.na(payroll_department)),
    mismatches = sum(department_match == "Mismatch", na.rm = TRUE),
    .groups = "drop"
  ) |>
  arrange(desc(mismatches), department)

write_csv(department_reconciliation_summary, "outputs/headcount_reconciliation_summary.csv")

Budget vs Actual Workflow

Situation
Finance needs a monthly budget-versus-actual view by cost center, with variance and variance percentage.

What Usually Happens In Excel
You open budget and actual files side by side, line up cost centers, repair formatting differences, use SUMIFS, then build a variance column and hope the formulas all extended correctly.

Cleaner R Workflow
Import both files. Clean names. Standardize cost-center labels and month fields. Join budget to actuals. Calculate variance and variance percentage. Rank the largest overruns. Export for leadership review.

Recommended Core Functions
read_excel(), read_csv(), clean_names(), mutate(), left_join(), group_by(), summarise(), arrange()

This is one of the most common “looks simple, eats a morning” workflows in corporate finance. The gain comes from making cost center logic, month alignment, and variance math visible all at once.

library(readxl)
library(readr)
library(dplyr)
library(stringr)
library(lubridate)
library(janitor)

budget <- read_excel("inputs/budget.xlsx", sheet = "Budget") |>
  clean_names() |>
  mutate(
    cost_center = str_to_upper(str_trim(cost_center)),
    report_month = floor_date(mdy(month), unit = "month"),
    budget_value = replace_na(budget_value, 0)
  )

actuals <- read_csv("inputs/actuals.csv") |>
  clean_names() |>
  mutate(
    cost_center = str_to_upper(str_trim(cost_center)),
    report_month = floor_date(mdy(transaction_date), unit = "month"),
    actual_value = replace_na(actual_value, 0)
  )

budget_vs_actual <- actuals |>
  group_by(report_month, cost_center) |>
  summarise(actual_value = sum(actual_value, na.rm = TRUE), .groups = "drop") |>
  left_join(
    budget |>
      group_by(report_month, cost_center) |>
      summarise(budget_value = sum(budget_value, na.rm = TRUE), .groups = "drop"),
    by = c("report_month", "cost_center")
  ) |>
  mutate(
    budget_value = replace_na(budget_value, 0),
    variance = actual_value - budget_value,
    variance_pct = if_else(budget_value == 0, NA_real_, variance / budget_value)
  ) |>
  arrange(desc(variance))

write_csv(budget_vs_actual, "outputs/budget_vs_actual_summary.csv")

Weekly KPI Reporting Pattern

Situation
A recurring weekly KPI file arrives from operations or reporting. The same columns need cleaning every time, and the same summary gets rebuilt every week.

What Usually Happens In Excel
You delete empty rows, trim labels, fix dates, rebuild a pivot table, copy the result into another workbook or PowerPoint slide, and repeat next week.

Cleaner R Workflow
Treat the KPI file as a standing pipeline. Import, clean names, trim text, handle missing values, summarize by site or region, export the table, save the plot.

Recommended Core Functions
read_csv(), clean_names(), str_trim(), replace_na(), group_by(), summarise(), write_csv()

This is the kind of workflow that becomes more valuable the less dramatic it feels. If the same KPI file keeps arriving, the right outcome is not excitement. It is calm repetition.

library(readr)
library(dplyr)
library(stringr)
library(tidyr)
library(janitor)

weekly_kpi <- read_csv("inputs/weekly_kpi_export.csv") |>
  clean_names() |>
  mutate(
    region = str_to_title(str_trim(region)),
    site = str_trim(site),
    metric_name = str_trim(metric_name),
    metric_value = replace_na(metric_value, 0)
  ) |>
  distinct(report_date, region, site, metric_name, .keep_all = TRUE)

weekly_kpi_summary <- weekly_kpi |>
  group_by(report_date, region) |>
  summarise(
    total_metric_value = sum(metric_value, na.rm = TRUE),
    records = n(),
    .groups = "drop"
  ) |>
  arrange(report_date, region)

write_csv(weekly_kpi_summary, "outputs/weekly_kpi_summary.csv")

Department Summary Pipeline

Situation
You need one clean departmental summary for headcount, spend, attrition, or workload, often by month or quarter.

What Usually Happens In Excel
You filter a large table, build a pivot, manually rename the values area, adjust sorting, and copy the output into a slide where the formatting never quite holds.

Cleaner R Workflow
Import the cleaned source file, standardize department values, group by department and time period, calculate the needed totals or counts, sort, and export.

Recommended Core Functions
read_csv(), mutate(), str_trim(), group_by(), summarise(), arrange()

A department summary is often one of those tasks that looks too small to deserve a script until you realize you have built it eighteen times already.

library(readr)
library(dplyr)
library(stringr)
library(lubridate)
library(janitor)

department_metrics <- read_csv("inputs/department_metrics.csv") |>
  clean_names() |>
  mutate(
    department = str_to_title(str_trim(department)),
    report_month = floor_date(mdy(report_date), unit = "month"),
    metric_value = replace_na(metric_value, 0)
  )

department_summary <- department_metrics |>
  group_by(report_month, department) |>
  summarise(
    total_metric_value = sum(metric_value, na.rm = TRUE),
    records = n(),
    .groups = "drop"
  ) |>
  arrange(report_month, desc(total_metric_value))

write_csv(department_summary, "outputs/department_summary.csv")

Payroll Cleanup Workflow

Situation
Payroll sends an export with duplicate employee IDs, ugly department values, missing manager names, and dates that arrive half as text and half as actual dates.

What Usually Happens In Excel
You delete rows, use helper columns, patch categories, repair dates manually, then save a “clean” version that no one can fully explain next week.

Cleaner R Workflow
Import the export. Clean names. Trim and standardize key fields. Replace missing values intentionally. Convert dates explicitly. Remove duplicates using employee and pay period as a rule. Save the clean file.

Recommended Core Functions
read_excel(), clean_names(), str_trim(), replace_na(), mdy() or dmy(), distinct(), write_csv(), write.xlsx()

This is where many spreadsheet users first feel the emotional payoff of a script. The file is still ugly. You are simply no longer negotiating with it by hand.

library(readxl)
library(dplyr)
library(stringr)
library(lubridate)
library(tidyr)
library(janitor)
library(readr)
library(openxlsx)

payroll_raw <- read_excel("inputs/payroll_extract.xlsx") |>
  clean_names()

payroll_clean <- payroll_raw |>
  mutate(
    employee_id = str_trim(employee_id),
    department = str_to_title(str_trim(department)),
    location = str_trim(location),
    manager_name = replace_na(manager_name, "Unassigned"),
    pay_date = mdy(pay_date),
    pay_period = floor_date(pay_date, unit = "month"),
    bonus_amount = replace_na(bonus_amount, 0)
  ) |>
  distinct(employee_id, pay_period, .keep_all = TRUE)

payroll_validation <- payroll_clean |>
  summarise(
    clean_rows = n(),
    total_base_pay = sum(base_pay, na.rm = TRUE),
    total_bonus = sum(bonus_amount, na.rm = TRUE)
  )

write_csv(payroll_clean, "outputs/payroll_clean.csv")
write.xlsx(
  list(
    payroll_clean = payroll_clean,
    validation = payroll_validation
  ),
  file = "outputs/payroll_clean.xlsx"
)

Executive Slide Preparation Pattern

Situation
The summary is complete, but it still needs to become something a senior audience can read quickly.

What Usually Happens In Excel
You build a chart, fight the default formatting, resize it for PowerPoint, and then explain verbally what the chart should have shown more clearly.

Cleaner R Workflow
Start from the trusted summary table. Order the categories. Build one clear chart. Use a title that states the message. Export at slide-ready resolution.

Recommended Core Functions
ggplot(), geom_col(), labs(), theme_minimal(), ggsave()

This pattern is not about design vanity. It is about making the output carry more of the conversation so you do not have to rescue the slide with explanation.

library(readr)
library(dplyr)
library(forcats)
library(ggplot2)
library(scales)
library(openxlsx)

regional_gap <- read_csv("inputs/regional_gap_summary.csv") |>
  mutate(region = fct_reorder(region, total_variance))

gap_chart <- ggplot(regional_gap, aes(x = region, y = total_variance)) +
  geom_col(fill = "steelblue") +
  labs(
    title = "Southeast And North Require Immediate Revenue Attention",
    subtitle = "Cumulative variance against target by region",
    x = NULL,
    y = "Variance"
  ) +
  scale_y_continuous(labels = label_dollar()) +
  theme_minimal(base_size = 12) +
  theme(
    panel.grid.minor = element_blank(),
    legend.position = "none"
  )

ggsave(
  filename = "outputs/regional_gap_chart.png",
  plot = gap_chart,
  width = 8,
  height = 5,
  dpi = 300
)

write.xlsx(regional_gap, "outputs/regional_gap_slide_support.xlsx")

A.4 Monday Morning Rescue Card

ImportantBefore You Open Excel Again

Ask these six questions first.

  • Is this task repeated weekly or monthly?
  • Will someone ask for this same summary again?
  • Are multiple files involved?
  • Are formulas breaking, drifting, or being copied down manually?
  • Is manual cleaning taking more than 20 minutes?
  • Should this be an R workflow instead?

If the answer to three or more is yes, you are probably not looking at a spreadsheet task anymore.

You are looking at a workflow that wants a script.

Fast Response Scaffold

Use this when the request is urgent and your brain needs a clean starting point.

library(readr)
library(readxl)
library(dplyr)
library(stringr)
library(tidyr)
library(janitor)
library(openxlsx)

ops_kpi_raw <- read_csv("inputs/weekly_ops_kpi.csv") |>
  clean_names()

region_lookup <- read_excel("inputs/finance_region_lookup.xlsx", sheet = "Regions") |>
  clean_names()

glimpse(ops_kpi_raw)
glimpse(region_lookup)

ops_kpi_clean <- ops_kpi_raw |>
  mutate(
    report_date = as.Date(report_date),
    region_code = str_trim(region_code),
    branch_name = str_to_title(str_trim(branch_name)),
    kpi_name = str_trim(kpi_name),
    kpi_value = replace_na(kpi_value, 0)
  )

regional_kpi_summary <- ops_kpi_clean |>
  left_join(region_lookup, by = "region_code") |>
  mutate(
    region = replace_na(region, "Unmapped Region")
  ) |>
  group_by(report_date, region) |>
  summarise(
    total_kpi_value = sum(kpi_value, na.rm = TRUE),
    branches_reported = n_distinct(branch_name),
    .groups = "drop"
  ) |>
  arrange(report_date, region)

write_csv(regional_kpi_summary, "outputs/regional_kpi_summary.csv")
write.xlsx(regional_kpi_summary, "outputs/regional_kpi_summary.xlsx")

# Save this script before closing the session so the next urgent request
# starts from a workflow, not from memory.

Last Quiet Reminder
